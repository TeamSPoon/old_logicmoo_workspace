GIPO(Graphical Interface for Planning with Objects) is an experimental GUI and tools environment for building planning domain models. It was written by Ron Simpson and Weihong Zhao of the Univ of Huddersfield as one output of the PLANFORM project from the Univ of Huddersfield, UK, with partners at the Univ of Salford and Univ of Durham.

GIPO is under continuous development, with a final version to appear in 2002. As it is the result of a research project, it is free. How to get GIPO?

Rationale

  1. Planning Domain Models are hard to design, write, debug, maintain - even for experts. The process of encoding is laborious. Bugs are of various types and can lurk in models for a long time. Presently, many domain model authors use planners themselves to try to develop and debug a domain model. Planners have not generally been designed for this purpose,

  2. A notation that is good for encoding and validation may _not_ be good for use with a particular class of planners. There is a need for translators from the encoding language to the input language for a taget planner. Further, with an increase in the number of useful tools and techniques surrounding planning, there is a need to integrate these into a tools environment.

  3. As planners and planning applications become larger, the problems of engineering planning domain models become more accute. There is a need to research into engineering environments and explore their synergy with general purpose planners.

Basic Functions

GIPO allows a user to create new domain models or import and change old ones. It features:

  1. Syntax and Semantic checks for individual components of a model

  2. Semantic cross checking between components

  3. Graphical construction of domain operators

  4. Import/Export facilities for PDDL models

  5. a planner stepper

  6. a plan animator

  7. a random task generator
