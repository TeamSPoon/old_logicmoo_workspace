Installation of GIPO? 

INSTALLATION Prerequisites

You need Java installed on your machine. GIPO is known to work with Java 1.4.2. It will not currently work with Java 1.5. The appropriate Java executable must be on your PATH.

Installation Windows Platforms

  1. Copy gipo<version>.zip file to destination directory e.g c:\gipoV3
  2. Unzip gipo-Win32-v3.?.zip note this will create a subdirectory gipo and and leave this file and the batch file runGIPO.bat in the top level directory.
  3. Edit runGIPO change the variable GIPO to reflect your installation directory. e.g. SET GIPO=c:\gipoV3\gipo NOTE this path must end in the gipo directory.
  4. If necessary add an appropriate Java bin directory to the PATH variable of runGIPO batch file.
  5. Make a shortcut to the runGIPO batch file or just run it from a dos/command window
  6. Enjoy

Installation Unix/Linux

  1. Copy gipo<architecture>-v3.?.tar.gz file to destination directory e.g /local/gipoV3
  2. Unzip/tar note this will create a subdirectory gipo and leave this file and the shell script gipo-<architecture> in the top level directory.
  3. Edit gipo-linux (or gipo-sun) change the variable GIPOHOME to reflect your installation directory. e.g. GIPOHOME=${HOME}/gipoV3/gipo NOTE this path must end in the gipo directory.
  4. Edit JAVA variable to reflect the location of the java runtime for java version 1.4.2 on your local machine.
  5. Move this shell script file to somewhere on your PATH or just run it from the current directory
  6. Enjoy

Source Build

The source is distributed with the assumption that you will be compiling using Linux, but the instructions should work with other platforms though you will need the relevant binaries for the Sicstus Prolog run time system and you may need to adjust the classpath for Java.

The code should compile with versions 1.4.2_(any) of the Java SDK it will not compile with Java 1.5

The distribution uses JGraph, JGraphAddons, Java_cup and Sicstus Prolog Java/Prolog interface classes. Look at the relevant licenses below.

To change the Ocl parser wou will need to download full versions of java_cup and JLex.

To compile uncompress the source distribution and from the gipo directory give the command

javac -classpath .:jgraph.jar:jgraphaddons.jar:java_cup.jar:lib/sicstus3.11.1/bin/jasper.jar jplan/top/OclEd.java

To run the created executable edit the gipo-src shell script to reflect the location of the distribution in your file system.
