PURPOSES:
Create and edit tasks.

 

 

BASIC REQUIREMENTS:

 

 

States must be checked by click the menu item "States" from the function "Validation" (use to be "Semantic Checks") on the menu bar.

 

 

TO CREATE A NEW TASK:

 

 

1. Click on the button "New Task", an input dialog box will appear. Type in a name to represent the new task.


 2. Expend the tree to find the object, click on the object, all related sub states for this object will be partially instantiated and displayed on the "states.." window.

 3. Click the state to bring it to the editing window. Click on the varible will highlight the varible with green underline, in the mean time any associated varibles (with same sort) will be highlighted with red underline. Right click on the green underlined varible will allow user to instantiate the varible by selecting an object from the popup menu. Right click on a predicate (rather than any green underlined varibles) will allow user to delete the predicate;

 4. Click button "Add Initial" or "Add Goal" to add the current edited state as one of the initial or goal states.
 
 

TO EDIT AN EXISTING TASK

 

 

5. Select the task from the task combo box on the right hand side of the window.


 6. Click on the "Inital State" window or "Goal State" window to bring the relavant state to the editing window. Follows the rule described at step 3 to edit the state to the user's desire.

 7. Click the button "Undate" to commit the changes. Be noticed that to add the task permanently to the domain, click on the button "Commit" on the top toolbar. Click on the button "Restore" will remove any changes since last click on the button of "Commit".
 
 

TO DELETE A TASK

 

 

8. select a task as described in step 5. Click on the button "Delete" to comfirm removal of current task.
