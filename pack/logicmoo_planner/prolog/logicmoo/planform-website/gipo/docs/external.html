PURPOSES:

 

 

GIPO allows for the integration of third party planning systems.

 

 

BASIC REQUIREMENTS:

 

 

1. The planner should process PDDL and be capable of dealing with the following requirements :strips :equality :typing and for domains expressed in gipo with conditional tansitions :conditional-effects.


 2. The planner should be capable of being run from the command line. It should accept command line parameters to specify the two files containing the domain definition and the task to be processed. The planner should write its output to the standard output stream.

PROCEDURE FOR INTEGRATING THE PLANNER:

 

 

Edit the file gipo.properties in the jplan directory of the GIPO distribution. Add the name of the planner (single symbol) to the end of the line beginning:

         ocled.planner.algs
Using the symbol you have choosen for the planner name add an entry for each of the folowing property lines:
          ocled.description.<planner name>=<text description of the planner system>
          ocled.condeffects.<planner name>=<yes or no>
          ocled.hierarchical.<planner name>=no
          ocled.hierarchicalsorts.<planner name>=yes
          ocled.callmethod.<planner name>=shell
          ocled.command.<planner name>=<command line>
In the above the
(1) text dscription may be a single line of text of any length describing the planning sysyem.
(2) the line for condeffects shoud be set to yes if the planner supports conditional effects otherwise set it to no.
(3) The command line should be the command line, including the command name, required to run the planner. Two escape sequences should appear in this line %d to mark the position where the name of the domain definition file occurs in the command line and %t to mark where the name of the task file occurs. In the command line the planner command must be preceded with a full path name except where the command exists on the PATH defined for the user of GIPO.
 
INTEGRATING THE PLANNER OUTPUT WITH THE ANIMATOR:

 

 

The animator can display the output of an external planner if it conforms to one of two formats. In bothe case to planner must output a fully ordered plan as a list, one per line, of operator names followed by the instantiated values of the operator parameters in the order given in the operator definition.


Standard form

TASK <task number>
SOLUTION
<list of steps, first to last.>
END FILE

In this form any text can appear before the first processed line beginning TEXT. Each step must have the form
<operator name>(<comma seperated list of instantiated parameters>)

Alternative form

step 0: <plan step 0>
         1: <plan step 1>
          ...
         n: <plan step n>

In the above any text not containing the word step can appear befort the line containing the key word. step. Any text can follow the list of stems except that the line following cannot start with the consequetive step number. Each step has the form
<operator name><space seperated list of instantiated parameters>
 

TO RUN THE PLANNER:

 

 

Simply select the planner name from the Algorithm dialog box of GIPO in the same way as you run an internal planner.
