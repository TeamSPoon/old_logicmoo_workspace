TO CREATE A NEW OPERATOR

 

 

1. click on the "New" button on the top toolbar or the menu bar (Operator), a green circle (labeled as "Action ID") will appear on the editing canvas to represent the head (name) of the operator.


 2. double click on this circle object to bring up a dialog box to rename the operator to the user's desire.

 3. click on the radio button "Add Prevail" from buttom toolbar to set the editing mode to "Adding a prevail condition for this new operator", then select the right state by clicking on it from the Substate List window on the left.

 4. click on the radio button "Add Necessary", then select a state from the substate list window, the left hand side statement of the necessary condition will appear on the canvas as a gray round rectangle, in the meantime, the right hand side statement appears as an empty round rectangle with flashing edges waiting for the user's input. The content of "Substate List" window has been updated to the states related to the same sort user just picked. Clicking on one of the state in the substate list window will specify the right hand side of the necessary condition and stop the flashing effect.

 5. to build a conditional change require the same operation as above NO 4 after the function mode is set to "Add Conditional" (bottom toolbar).

 6. click on the "Add" button on the top toolbar to save this operator to the "Operators list". Be noticed that to save the operator to the domain for the other tools to use, "Commit" button needs to be clicked before leave this transition window.
 
 

TO VIEW/EDIT AN EXISTING OPERATOR

 

 

7. double click on the new operator from the "Operators List", the newly created operator will be reorganised on the canvas.


 8. right click on any rectangles (not the operator head/name - the circle) to bring up a popup menu with all the static predicates. To add the necessary static predicate, just click on right item on the popup menu.

 9. click on the "Edit Mode" on the top toolbar to freeze the all the shapes on the canvas. The system reaches a situation that all the varibles of this operator can now be edited. "Zoom In" and "Zoom Out" function will help user to adjust the viewing scale.

 10. click the varible which needs to be renamed. It will be highlighted with green underline, all other related (with same sort) varibles will also be highlighted but with red underline. Right click on the green underlined varible will allow user to rename it. Right click on the red underlined varible will allow user to specify the relationship between the current red underlined varible and the green underlined varible. Right click on the begining part of a predicate will allow user to remove this predicate from the condition/shape.

 11. click on the "Update" button (not the "Add" button) to update current edited operator.
 
 

TO DELETE AN OPERATOR

 

 

12. click on the right operator from the "Operators List" window, then click on the button "Delete" on TOP toolbar. The button "Delete" on the bottom toolbar is for deleting a sigle shape on the canvas.

 

 

OTHER FUNCTIONS


 13. All shapes and link lines support the common functions such as click select, drag select, delete, drag/move, resize, reshape, undo/redo etc.

 14. Any small windows, toolbars can be hidden by selecting the right mode in "View" functions of the menu bar.

 15. The graphics of the operator can be saved as pure graphical file and can be printed out to a ps file or a printer.
