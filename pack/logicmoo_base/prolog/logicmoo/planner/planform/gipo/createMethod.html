To Create A Compound Operator


  1. Click the "Compound Transitions" button on the toolbar to bring up the "Compound Transitions" window (Figure 1).
  2. Click on the "New" button on the top toolbar or the menu bar (Operator), a pale blue circle (labeled as "Action ID") will appear on the editing canvas to represent the head (name) of the compound operator.
  3. Double click on this circle object to bring up a dialog box to rename the compound operator.
  4. Click on the radio button "Add Pre-conditions", then select the proper state from the substate list window. Click on the canvas to bring up a brown square standing for Pre-conditions.
  5. Click on the radio button "Add Index", then select the proper state from the substate list window. Click on the canvas, the left hand side statement of the condition will appear on the canvas as a gray blue round rectangle, in the meantime, the right hand side statement appears as an empty round rectangle with flashing edges waiting for the user's input (Figure 1). The content of "Substate List" window has been updated to the states related to the same sort user just picked. Clicking on the proper state in the substate list window will specify the right hand side of the necessary condition and stop the flashing effect.
  6. Click on the radio button "Add Statics", then select one or more statics from the static list window on the left hand side of the window. Note that 'CONTROL' key with mouse click will allow user to multi-select, 'SHIFT' key with mouse click will allow user to select a range (Figure 2). After specify all proper statics, click on the canvas, a light gray round square containing all selected static predicates will appear on the canvas.
  7. Change to edit mode by selecting the check box "Edit Mode". Drawing canvas will change to dark blue. The system reaches a situation that all the variables of this operator can now be edited. "Zoom In" and "Zoom Out" function will help user to adjust the viewing scale.
  8. Click on the variable which needs to be edited. It will be highlighted with green underline, all other related (with same sort) variables will also be highlighted but with red underline. Right click on the green underlined variable to rename. Right click on the red underlined variable will allow user to specify the relationship between the current red underlined variable and the green underlined variable (Figure 3).
  9. Right click on the head shape of the compound operator. A popup menu will appear (Figure 4). Click 'Show decomposition', the head shape will grow to the window size, then inside the head shape a decomposition window/canvas will be opened with animation (Figure 5). Note that the statics list window is replace by the operators list window (Figure 5).
  10. Right click on the decomposition canvas, all possible functions are listed in a menu (Figure 5).
  11. To put a existing compound operator in the decomposition canvas, just simply drag the desired compound operator from the compound operators list window (Figure 6). In the same way, drag an existing operator from the operators list window on the left hand side. The special 'achieve' state (oclSS) can also be formed by dragging the appropriate states from the states list window. Right click on the decomposition canvas, then select the 'temporal constraints' from the popup menu(Figure 7).
  12. Select the shapes which has a precedent relationship to form the temporal constraint (Figure 8).
  13. Right click on the shapes, object property menu is shown (Figure 9). Click the menu, the property of the selected object (compound operator/operator/oclSS) will be displayed at another canvas registered as another tab pane (Figure 10). At this level, the shapes are not editable but viewing contents only. Zoom in/out button on the toolbar is active to help adjusting the view port size. This compound operator can be decomposed as well (Figure 10), but compound operators/operators can not be dragged onto the lever level decomposition canvas to form the shape, i.e. apart from the first level, any other levels of decomposition are not editable.
  14. In the achieve clause property window, editing is allowed to change the variables' name, etc (Figure 11).
  15. To go back the undecomposed mode, right click on the head shape (not inside the decomposition canvas), a menu will appear (Figure 12). Select the menu, the decomposition window will close, then the head shape will shrink to its normal size as in Figure 1.
  16. Click on the "Add" button on the top toolbar to save this operator to the "Compound operators list".
  17. Be noticed that to save the compound operator to the domain for the other tools to use, "Commit" button needs to be clicked before leave this compound transition window.
  18. To view an existing compound operator, double click on the compound operator from the "Compound operators list" on the right hand side of the window. Follow the same procedure discussed above to edit.
